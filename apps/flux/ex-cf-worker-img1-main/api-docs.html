Overview

This API provides access to two state-of-the-art AI image generation models: FLUX-1-schnell and Leonardo Phoenix 1.0. Generate high-quality images from text descriptions with simple REST API calls.

Quick Start

Generate your first image in seconds:

curl -X POST https://flux-image-generator.example.workers.dev/api/flux \
  -H "Content-Type: application/json" \
  -d '{"prompt": "a beautiful sunset over mountains"}' \
  --output image.png

Base URL

https://flux-image-generator.example.workers.dev

Available Endpoints

  • GET /api/info - Get API information and available models
  • POST /api/flux - Generate image with FLUX model
  • POST /api/phoenix - Generate image with Phoenix model

Available Models

FLUX-1-schnell Fast

Optimized for speed with good quality. Perfect for quick iterations and real-time applications.

  • 1-8 inference steps
  • ~2-3 seconds generation time
  • Supports aspect ratios
  • Lower cost per image
  • PNG output format

Phoenix 1.0 Quality

High-quality image generation with advanced control. Ideal for professional and artistic use.

  • 1-50 inference steps
  • Guidance scale control
  • Negative prompts
  • Custom dimensions up to 2048px
  • JPEG output format
  • Seed support for reproducibility

Authentication

Currently, the API is open and does not require authentication. This may change in the future.

Note: CORS is enabled for all origins. The API accepts requests from any domain.

FLUX Model API

POST /api/flux

Request Parameters

Parameter Type Required Default Description
prompt string Yes - Text description of the image to generate
steps integer No 4 Number of inference steps (1-8)
aspectRatio string No "1:1" Aspect ratio: "1:1", "16:9", "9:16", "4:3", "3:4"

Example Request

curl -X POST https://flux-image-generator.example.workers.dev/api/flux \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a cyberpunk city at night with neon lights",
    "steps": 4,
    "aspectRatio": "16:9"
  }' \
  --output cyberpunk.png
const response = await fetch('https://flux-image-generator.example.workers.dev/api/flux', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    prompt: 'a cyberpunk city at night with neon lights',
    steps: 4,
    aspectRatio: '16:9'
  })
});

const blob = await response.blob();
const imageUrl = URL.createObjectURL(blob);
document.getElementById('image').src = imageUrl;
import requests

response = requests.post(
    'https://flux-image-generator.example.workers.dev/api/flux',
    json={
        'prompt': 'a cyberpunk city at night with neon lights',
        'steps': 4,
        'aspectRatio': '16:9'
    }
)

with open('cyberpunk.png', 'wb') as f:
    f.write(response.content)

Response

Success 200
Binary image data (PNG format)

Response Headers:
Content-Type: image/png
X-Model: flux-1-schnell
X-Steps: 4
X-Width: 1344
X-Height: 768

Phoenix Model API

POST /api/phoenix

Request Parameters

Parameter Type Required Default Description
prompt string Yes - Text description of the image to generate
num_steps integer No 25 Number of diffusion steps (1-50)
guidance number No 2 Guidance scale (2-10). Higher = more prompt adherence
width integer No 1024 Image width in pixels (max 2048)
height integer No 1024 Image height in pixels (max 2048)
seed integer No random Random seed for reproducible results
negative_prompt string No - What to exclude from the generated image

Example Request

curl -X POST https://flux-image-generator.example.workers.dev/api/phoenix \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a majestic phoenix rising from flames",
    "num_steps": 30,
    "guidance": 7.5,
    "width": 1920,
    "height": 1080,
    "negative_prompt": "blurry, low quality, distorted"
  }' \
  --output phoenix.jpg

Error Handling

Error Response Format

{
  "error": "Error message description"
}

Common Error Codes

Status Code Description Solution
400 Bad Request - Missing required parameters Ensure prompt is provided
405 Method Not Allowed Use POST method for image generation
500 Internal Server Error Model error or temporary issue, retry request
Error 400
{
  "error": "Prompt is required"
}

Rate Limits & Best Practices

Current Limits

  • No hard rate limits currently enforced
  • Recommended: Max 10 concurrent requests
  • Average response time: 2-5 seconds depending on model and parameters

Best Practices

  • Optimize prompts: Be specific and descriptive for better results
  • Use appropriate steps: More steps = better quality but slower
  • Cache results: Store generated images to avoid redundant API calls
  • Handle errors gracefully: Implement retry logic with exponential backoff
  • Choose the right model: Use FLUX for speed, Phoenix for quality

Cost Optimization Tips

  • Start with fewer steps and increase if needed
  • Use smaller dimensions for drafts
  • Test prompts with FLUX before using Phoenix
  • Use seeds for consistent variations
